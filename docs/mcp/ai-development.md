# AI Development Guide

Руководство по разработке на платформе Solafon с помощью AI-ассистентов.

## Зачем использовать AI для разработки на Solafon?

1. **Быстрый старт** — AI генерирует готовый код бота за секунды
2. **Доступ к документации** — AI знает все API и может подсказать нужный endpoint
3. **Отладка** — AI проверяет вебхуки, токены и статус API в реальном времени
4. **Итеративная разработка** — описываете фичу словами, AI реализует

## Рабочий процесс

### 1. Подключите MCP Server

См. [Setup Guide](setup.md) для настройки Claude Desktop, Cursor или VS Code.

### 2. Создайте бота через AI

Просто скажите:
> "Создай мне Solafon бота на Node.js с вебхуком, который отвечает на команды /start и /help, а обычные сообщения пересылает в OpenAI GPT-4"

AI автоматически:
- Прочитает документацию (`solafon_read_docs`)
- Сгенерирует код (`solafon_scaffold_bot`)
- Настроит команды (`solafon_set_commands`)
- Установит вебхук (`solafon_set_webhook`)

### 3. Тестируйте в реальном времени

> "Проверь, есть ли новые сообщения для моего бота"

AI вызовет `solafon_get_updates` и покажет результат.

> "Отправь тестовое сообщение пользователю с ID 5"

AI использует `solafon_send_message`.

### 4. Отлаживайте проблемы

> "Мой бот не получает сообщения. Помоги разобраться."

AI выполнит комплексную диагностику:
- Проверит здоровье API
- Проверит токен
- Проверит конфигурацию вебхука
- Посмотрит pending-сообщения

## Лучшие практики

1. **Начинайте с шаблона** — используйте `solafon_scaffold_bot` как основу
2. **Инкрементальная разработка** — добавляйте фичи по одной
3. **Тестируйте сразу** — используйте `solafon_send_message` и `solafon_get_updates` для тестов
4. **Используйте auto-response** — простые команды не требуют сервера, настройте response в `solafon_set_commands`
5. **Храните токен безопасно** — всегда через переменные окружения, никогда в коде

## Архитектурные паттерны

### Простой бот (auto-response)
Для ботов с фиксированными ответами — настройте команды через `solafon_set_commands` с полем `response`. Сервер не нужен.

### Polling бот
Для простых ботов — запрашивайте новые сообщения через `GET /bot/getUpdates` каждые 2-5 секунд.

### Webhook бот
Для production — настройте HTTPS-сервер и зарегистрируйте URL через `POST /bot/setWebhook`.

### AI бот
Подключите OpenAI/Claude к вашему боту для интеллектуальных ответов. Шаблон `node-ai-bot` — готовый стартер.
